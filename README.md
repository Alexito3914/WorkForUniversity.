# WorkForUniversity.
2 Version
Autenticación y roles
Ahora el sistema tiene login y logout. Nadie puede acceder sin identificarse. Cada usuario tiene un rol (Decanato, Profesor, Estudiante, IT o Consultor) y según ese rol ve unas opciones u otras. Por ejemplo, solo el Decanato puede aprobar horarios, y un profesor solo puede tocar su propia disponibilidad.
Interfaz visual real
La V1 solo tenía una home con contadores y un dashboard vacío. La V2 añade páginas funcionales: una lista de todos los horarios con su estado, una vista de detalle con la tabla lunes–viernes donde se ven las clases por franja horaria, y una página de titulaciones con sus cursos y asignaturas. Todo filtrable y con la identidad visual de la UCJC.
Gestión de horarios y disponibilidad
Se puede crear un horario desde la web, pasarlo por el flujo de aprobación (Borrador → Revisión → Aprobado o Rechazado), y reabrirlo si hace falta. Los profesores pueden entrar y marcar en qué franjas están disponibles o bloqueados.
